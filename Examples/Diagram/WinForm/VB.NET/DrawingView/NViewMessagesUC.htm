This example demonstrates the view messages, which appear when certain operations cannot be completed or commited. This is in most cases caused when the elements to which you try to apply a certain action do not have the appropriate permission or some other properties of the object prohibit this action from being executed.

In this example the red shape does not have a Change Style permission. To explore the behavior of the component when transactions cannot be completed do the following:

  1. Select the green and the red shapes
  2. Right click on the red or green shape to show a context sensitive menu
  3. Go to Format - Fill style... and click on it - this will show the fill style editor.
  4. Select a fill style and click the OK button
Only the green shape will change its appearance and a designer message will pop up indicating that the transaction was not completed. Note that not completed transaction are still committed - e.g. not completed means that only parts of the transaction failed.

Most of the methods, which record transactions accept an additonal rollbackOnFail argument, which indicates whether the entire transaction should fail, if only a part of it failed.
